feat: deletable documents (all versions + comments, owner-only)

Uploaded documents can now be deleted from the project page.
No schema changes — comment cleanup rides on the existing
ON DELETE CASCADE foreign key.

worker/index.js
- NEW DELETE /api/documents/:id — deletes one document version.
  Project owners (or internal) only. Order matters: D1 row first
  (comments cascade via FK), then the R2 object; an R2 delete
  failure is swallowed since an orphaned object is harmless while
  a dangling DB row would be a broken document.

project.html
- 削除 / Delete button on each document row (owners only), with
  event.preventDefault/stopPropagation so it doesn't trigger the
  row's viewer link
- Deleting removes ALL versions of that filename (looped over the
  version list) after a confirm dialog stating the version count
  and that attached comments go with them
- Per-file result toasts with partial-failure reporting
